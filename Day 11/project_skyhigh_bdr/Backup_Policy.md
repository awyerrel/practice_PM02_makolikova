# План резервного копирования (Backup Policy)
**Компания:** SkyHigh Airlines
**Версия:** 1.0
**Дата:** 17.06.2026

## 1. ЦЕЛИ И ЗАДАЧИ

Обеспечить выполнение целевых показателей RPO и RTO для всех критических сервисов.

**Требования:**
- RPO ≤ 15 минут для всех систем
- RTO ≤ 4 часа для всех систем

**Ключевые показатели по компонентам:**

| Компонент | RPO | RTO | Обоснование |
|-----------|-----|-----|-------------|
| DB2 (Бронирование) | 1 минута | 15 минут | Потеря брони = потеря прибыли |
| PostgreSQL (Диспетчерская) | 5 минут | 1 час | Безопасность полетов |
| Redis (Кэш планов) | 1 минута | 30 минут | Критичен для диспетчеров |
| MSSQL (MRO) | 15 минут | 4 часа | Техобслуживание самолетов |
| S3 (Спутниковые данные) | 1 час | 8 часов | Метеопрогнозы |
| Kafka (События) | 5 минут | 1 час | Аналитика рейсов |

## 2. СТРАТЕГИЯ 3-2-1

**3 копии данных:**
1. Production данные (активная среда, us-east-1)
2. Локальный бэкап-сервер (NVMe RAID 10)
3. Облачное хранилище AWS S3 (eu-central-1)

**2 разных носителя:**
1. SSD/NVMe диски (для оперативных бэкапов)
2. Ленточная библиотека LTO-9 (для архивов)

**1 оффсайт копия:**
- AWS S3 в регионе eu-central-1 (Frankfurt)

## 4. ОБОСНОВАНИЕ ВЫБОРА ТЕХНОЛОГИЙ

**IBM DB2:**
- Прозрачное шифрование бэкапов (AES-256)
- HADR для автоматической репликации
- Archive Logs для PITR с точностью до секунды

**PostgreSQL:**
- WAL-архивация с минимальной нагрузкой
- pg_basebackup для горячего бэкапа
- Immutable Storage для защиты от шифровальщиков

**Redis:**
- RDB Snapshot для быстрого восстановления
- AOF для реал-тайм записи операций
- Redis Sentinel для автоматического failover

**AWS S3:**
- CRR для мгновенной репликации
- Glacier Deep Archive для экономии (96%)
- Object Lock для юридической защиты

## 6. ПОЛИТИКА ШИФРОВАНИЯ

**В состоянии покоя:** AES-256 для всех бэкапов
**В передаче:** TLS 1.3 для всех операций с S3
**Ключи:** AWS KMS с ротацией каждые 90 дней
**Защита от удаления:** S3 Object Lock на 7-30 дней
**MFA Delete:** Включено для всех бэкап-бакетов

## 8. МОНИТОРИНГ И КОНТРОЛЬ

**Ключевые метрики:**
- Успешность бэкапов ≥ 99.9%
- Время выполнения не должно превышать окно бэкапа
- Возраст последнего бэкапа не должен превышать RPO * 1.5

**Алерты:**
- Критический: Бэкап не выполнялся > RPO * 2
- Предупреждение: Свободное место < 20%
- Инфо: Бэкап выполнен с ошибками

**Профилактика:**
- Еженедельно: Проверка успешности всех бэкапов
- Ежемесячно: DR Drill в изолированной среде
- Ежеквартально: Инвентаризация оборудования
- Ежегодно: Полный аудит системы бэкапов
